# Small-Task Review Packet Retention Example

This low-risk documentation task does not use OpenSpec and does not invent C1-C4 stages.

Scope: update one local paragraph, run required validation, obtain the required final review, and make no git exit.

Final applicable gate: required final review.

After that review completes with no P0/P1, `BLOCKED-EVIDENCE`, evidence gap, correction, Owner decision, or later gate pending, the filled packets and generated prompts become eligible for non-destructive compaction. The reusable blank templates, original changed file, validation evidence, invocation identity, findings, complete reasoning, and evidence pointers remain durable.

If the same task later includes commit, push, or Owner-authorized publication, cleanup waits for actual-commit review, post-push status/review, or post-action closeout respectively. File deletion always requires separate exact-scope Owner authorization.

The filled files may therefore remain stored and accumulate after compaction eligibility; the rule controls active/retransmitted context rather than promising a storage cap.

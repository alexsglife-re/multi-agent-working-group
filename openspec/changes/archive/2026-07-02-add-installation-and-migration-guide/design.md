# Design: Installation And Migration Guide

## Guide Shape

Use a short, stable markdown guide at `docs/INSTALLATION.md`:

- prerequisites;
- local checkout usage;
- optional global skill sync;
- migrating to another machine;
- adopting in another project;
- validation commands;
- what does not transfer.

## Boundaries

Migration is file and documentation transfer only. It must not imply that role continuity, PM/Advisor agreement, Worker returns, validation freshness, commit/push authorization, CI/archive permission, or external-effect approval carries to another machine or project.

Concrete provider/model preferences should be recorded in Owner instructions, global/project memory, handoff/startup packets, or current verified records. The guide should describe that mechanism without naming a fixed model.
